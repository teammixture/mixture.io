Publishing & Hosting

UPDATE: 6th June 2014

We have now added support for publishing and hosting on Amazon S3 and some of the settings detailed in the videos below have now changed – please always refer to the changelogs and docs following an update of the applications.

Mixture has plenty of options when it comes to publishing and hosting your project, in this post we’ll take a closer look at each option.

GitHub pages

Static hosting from GitHub.

Details

  • Its Free.
  • Hosted on the GitHub infrastructure.
  • Git versioning by default.
  • Can take 10 minutes from publish to live.
  • Static sites only (HTML converted Mixture project only).
  • Requires a GitHub account.

You don’t need Git installed, Mixture has native support and makes it super easy to create and publish your project to GitHub.

The following short video demonstrates the process from setup to publish.

Mixture Hosting

Publish your project quickly and easily to Mixture.

Details

  • One click publish.
  • Unique URL or add your own domain.
  • Password protect.
  • Dynamic, its more than static the same dynamic templating features that work offline also work online.
  • Only changed files are published.
  • Mixture takes care of gzip compression and more.
  • Requires a pro account.

Build your project. One click publish. Simple.

The following short video demonstrates the process from setup to publish.

FTP

FTP your project anywhere.

Details

  • Use your own hosting and setup.
  • Support for both Mixture and Simple mode projects.
  • Only changed files are published.
  • Requires your own hosting / environment.

Add your FTP details to the project settings, select tools > FTP and Mixture will report how many files were published and if any were skipped.

We’re looking at additional options and improvements, if you would like us to support a particular hosting platform please let us know.
